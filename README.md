## About

**gl2** is a statically typed, single-pass, self-hosted systems programming language that compiles to x86_64 AT&T assembly.

## Features and architecture

- Single-pass compilation without an AST.
- Direct x86_64 assembly generation relying on GNU assembler and linker.
- Fully bootstrapped environment.
- Strict 64-bit alignment.

All data types in **gl2** operate on an 8-byte boundary. This eliminates type complexity and guarantees optimal memory access speeds on x86_64 platforms.

## Building

```bash
git clone https://github.com/zivadinovic/gl2
cd gl2/gl2_1_bs_sysv
chmod +x bootstrapit.sh
chmod +x compileit.sh
./bootstrapit.sh
```

To compile a source file, run the **compileit.sh** shell script like this:
```bash
./compileit.sh "input.gl2" "generated_dest.s" "generated_ascii_dest.s"
```
Where *generated_dest.s* is the desired path for the raw assembly output generated by the compiler, and *generated_ascii_dest.s* the path for the assembly file converted to standard single-byte character encoding.

## History

My journey to make a self-hosted compiler took over three and a half years of constant learning and exploration, ultimately culminating in this project.

> [!NOTE]
> The current version utilizes custom array length tracking (length stored at index `-1`) to manage memory allocations safely without a garbage collector.

## Performance and Known Limitations

- Excessive syscall usage: The compiler relies heavily on Linux system calls. Right now, the compiler makes a high volume of system calls during the compilation. These frequent calls create a noticeable bottleneck in compilation speed.
- Assembly comment shifting: Due to how the compilation handles assembly blocks, the assembly comments `asm {" ## a comment "}` placed at the global scope may be pushed into the prologue of subsequent function in the generated assembly output file.
